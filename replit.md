# SKYLAPSE - Camera Monitoring System

## Overview

SKYLAPSE is a multi-tenant SaaS platform for IP camera monitoring with automatic photo capture and on-demand timelapse generation. The system allows administrators to manage clients, locations, and cameras while clients can only view resources assigned to them.

**Core Features:**
- Multi-tenant architecture with admin and client roles
- IP camera management across multiple locations
- Automatic photo capture at configurable intervals
- On-demand timelapse video generation
- Real-time camera status monitoring

## User Preferences

Preferred communication style: Simple, everyday language.

## System Architecture

### Frontend Architecture
- **Framework:** React 18 with TypeScript
- **Routing:** Wouter for lightweight client-side routing
- **State Management:** TanStack React Query for server state and caching
- **UI Components:** shadcn/ui component library built on Radix UI primitives
- **Styling:** Tailwind CSS with custom theme variables supporting light/dark modes
- **Build Tool:** Vite with hot module replacement

The frontend follows a page-based structure under `client/src/pages/` with shared components in `client/src/components/`. Admin pages are grouped under `/admin/` routes. The application uses a sidebar layout with breadcrumb navigation for the admin interface.

### Backend Architecture
- **Framework:** Express.js with TypeScript
- **Database ORM:** Drizzle ORM with PostgreSQL
- **Authentication:** Replit Auth (OpenID Connect) with session storage in PostgreSQL
- **API Design:** RESTful endpoints under `/api/` prefix

The backend follows a layered architecture:
- `server/routes.ts` - API route definitions
- `server/storage.ts` - Data access layer implementing the IStorage interface
- `server/db.ts` - Database connection using node-postgres Pool

### Data Model
The schema defines five main entities:
- **clients** - Business customers using the platform
- **locations** - Physical sites with cameras
- **cameras** - IP camera configurations with connection details
- **captures** - Photo captures from cameras
- **timelapses** - Generated timelapse videos

All entities use UUID primary keys generated by PostgreSQL.

### Authentication Flow
- Replit Auth provides OpenID Connect authentication
- Sessions stored in PostgreSQL via connect-pg-simple
- Protected routes use `isAuthenticated` middleware
- User data synced to local users table on login

## External Dependencies

### Database
- **PostgreSQL** - Primary data store accessed via `DATABASE_URL` environment variable
- **Drizzle ORM** - Type-safe database queries with schema defined in `shared/schema.ts`

### Authentication
- **Replit Auth** - OpenID Connect provider (configurable via `ISSUER_URL`)
- **express-session** - Session management with PostgreSQL store
- Required environment variables: `SESSION_SECRET`, `REPL_ID`

### UI Framework
- **shadcn/ui** - Component library configured in `components.json`
- **Radix UI** - Accessible primitive components
- **Tailwind CSS** - Utility-first CSS with custom design tokens

### Build & Development
- **Vite** - Development server and production bundler
- **esbuild** - Server-side TypeScript bundling
- **tsx** - TypeScript execution for development

### Key NPM Packages
- `@tanstack/react-query` - Data fetching and caching
- `drizzle-orm` / `drizzle-zod` - Database ORM with Zod validation
- `wouter` - Lightweight React router
- `date-fns` - Date manipulation utilities
